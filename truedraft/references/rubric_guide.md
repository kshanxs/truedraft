# Rubric Guide

## Purpose
Interpret academic rubrics and marking criteria, then calibrate the writing pipeline to maximise marks in each graded dimension. Load this file when the user provides a rubric, marking scheme, or says "here are the criteria."

---

## How to Read a Rubric

When the user pastes a rubric or marking criteria:

### Step 1 — Extract Weighted Dimensions
Identify each graded category and its available marks or percentage.

| Dimension | Marks | Writing strategy |
|---|---|---|
| Content / Knowledge | high weight | depth of concept explanation, accuracy |
| Critical Analysis | high weight | evaluate, compare, question — don't just describe |
| Structure / Organisation | medium | follow the assignment structure precisely |
| Academic Style / Tone | medium | match the formality the marker expects |
| Referencing | medium | use the required citation style exactly |
| Word Count | constraint | stay within ±10% of the stated limit |

### Step 2 — Identify the Highest-Weight Dimension
Write primarily to earn marks in the highest-weighted category. If "Critical Analysis" is 40% of marks, every major paragraph should contain an analytical or evaluative move — not just explanation.

### Step 3 — Note Specific Requirements
Flag anything the rubric explicitly requires:
- "must include primary sources"
- "compare two theoretical frameworks"
- "use a real-world case study"
- "structured as: Introduction → Literature Review → Analysis → Conclusion"

These are non-negotiable. Include them even if they feel forced.

## Rubric Levels — How to Aim for the Top Band

Most rubrics have bands like: Fail / Pass / Credit / Distinction / High Distinction (or 1st / 2:1 / 2:2 / 3rd in UK systems).

Top-band writing typically requires:
- **Insight beyond the obvious** — not just explaining what, but *why it matters* and *what the limits are*
- **Engagement with complexity** — acknowledging counterarguments, edge cases, debates
- **Precise academic language** — field-specific terminology used correctly
- **Source integration** — citing evidence in support of *your* argument, not substituting for it
- **Structural coherence** — each paragraph clearly advances the overall argument

## Rubric Dimension Strategies

### Critical Analysis (most commonly underweighted by students)
Don't just describe. After every explanation, add one of:
- An evaluation: *"This is effective because... however, it is limited by..."*
- A comparison: *"Unlike X, Y achieves... at the cost of..."*
- A question: *"This raises the question of whether..."*
- An implication: *"If this holds true, then..."*

### Content / Knowledge
Cover all major subtopics implied by the assignment question. Use the research agent to ensure nothing important is missed.

### Structure / Organisation
If the rubric specifies sections (e.g., Introduction / Body / Conclusion, or IMRAD format), follow that structure precisely. Create a heading for each required section.

### Referencing
Load `references/citation_policy.md` and apply the required style using only user-provided sources.

## Writing to a Rubric — Workflow Addition

When a rubric is available, insert this step between the plan and draft stages:

**Rubric Mapping Step:**
1. List each rubric dimension and its weight
2. Note the top-band descriptor for each
3. For each, write one sentence describing how this assignment will achieve it
4. Pass this map to the draft stage as additional context

This ensures the draft is structured around marks, not just content.

---

## No Rubric Available?

If the user has no rubric, use these safe defaults appropriate to most university-level assignments:
- 40% critical analysis / argument
- 30% content accuracy and depth
- 20% structure and clarity
- 10% referencing
